YaMusicToTelegram
-скрипт, который следит за треком, играющим в Яндекс.Музыке на Windows, и публикует его в Telegram-канал: обложка, артист, название, альбом, прогресс-бар с таймером. При смене трека старый пост удаляется и публикуется новый. Отдельно обрабатываются пауза (публикуется своя картинка) и полное отсутствие воспроизведения.
Возможности
Автоопределение текущего трека через встроенный в Windows WinRT Media API (`GlobalSystemMediaTransportControlsSessionManager`) — без Python, без сторонних пакетов.
Обложка и точная длительность трека подтягиваются из каталога Яндекс.Музыки (`api.music.yandex.net`).
Три состояния поста в канале:
▶ Играет — обложка трека, прогресс-бар, таймер.
⏸ Пауза — своя картинка (`pause_cover.png`), без прогресс-бара.
🔇 Ничего не играет — та же картинка, что и для паузы.
Прогресс-бар обновляется редактированием существующего поста, а не пересозданием — с ограничением частоты, чтобы не словить от Telegram `429 Too Many Requests`.
Отдельный лог-файл событий (`track_events.log`) — старт трека, пауза, возобновление, сброс таймера.
Устойчивость к сети: таймауты на все HTTP-запросы, обработка `retry_after` от Telegram.
Отключение QuickEdit Mode консоли — случайный клик по окну больше не ставит скрипт на паузу.
Требования
Windows 10/11.
PowerShell 5.1 (встроенный в Windows) или PowerShell 7+.
Яндекс.Музыка (десктопное приложение или сайт/UWP — главное, чтобы сессия была видна в системном медиа-центре Windows, том самом, что показывает медиа-оверлей при Fn-клавишах).
Telegram-бот и канал, в котором этот бот — администратор с правом публикации.
Установка
Скачайте `YaMusicToTelegram.ps1` и `pause_cover.png` в одну папку.
Создайте бота через @BotFather и получите токен вида `123456789:AAA...`.
Добавьте бота администратором в канал (право «Публиковать сообщения» обязательно).
Узнайте `chat_id` канала — самый простой способ: перешлите любое сообщение из канала боту @userinfobot или @JsonDumpBot, либо откройте `https://api.telegram.org/bot<ТОКЕН>/getUpdates` после того, как в канал придёт любое новое сообщение. Для каналов `chat_id` обычно отрицательный и начинается с `-100`.
Впишите токен и `chat_id` в скрипт — см. раздел ниже.
Запустите:
```
   powershell -ExecutionPolicy Bypass -File YaMusicToTelegram.ps1
   ```
Куда вписывать данные пользователя
Все пользовательские настройки собраны в одном блоке `НАСТРОЙКИ` в самом начале файла, строки 21–33. Конкретно:
Строка	Что менять	Пример
22	`$TelegramBotToken` — токен вашего бота от BotFather	`$TelegramBotToken = "123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"`
23	`$TelegramChatId` — id канала, куда публиковать	`$TelegramChatId = "-1001234567890"`
26	`$AppNameFilter` — по этой подстроке скрипт ищет нужную медиа-сессию среди всех, что видит Windows (плеер, браузер и т.д.). Менять не нужно, если используете десктопное приложение Яндекс.Музыки	`"yandex"`
30	`$PauseImagePath` — путь к картинке для паузы/простоя. По умолчанию ищет `pause_cover.png` рядом со скриптом	своя картинка при желании
Остальные строки (25, 27–29, 31–32) — технические настройки (интервал опроса, размер обложки, таймауты, файл лога) с пояснениями прямо в комментариях рядом; трогать их не обязательно.
Важно: строки 22 и 23 — единственные два места, без заполнения которых скрипт не заработает.
⚠️ Перед публикацией на GitHub
В текущем файле в строках 22–23 сейчас стоят реальные боевые токен и chat_id, которыми пользовались при разработке. Публиковать файл в таком виде на GitHub нельзя — любой, кто увидит репозиторий, сможет полностью управлять вашим ботом (постить и удалять сообщения от его имени, менять описание и т.д.).
Перед коммитом:
Замените строки 22–23 на пустые плейсхолдеры:
```powershell
   $TelegramBotToken = "ВАШ_ТОКЕН_ОТ_BOTFATHER"
   $TelegramChatId   = "ВАШ_CHAT_ID"
   ```
Токен, который уже засветился (в этом файле/переписке), лучше отозвать через `@BotFather` → `/revoke` и выпустить новый — просто заменить значение в строке 22 недостаточно, если старый токен уже где-то виден.
Добавьте в `.gitignore` файлы, которые скрипт создаёт сам во время работы и которые не нужны в репозитории:
```
   last_message_id.txt
   track_events.log
   ```
Более безопасный вариант для собственного использования — не хранить токен в самом скрипте, а брать его из переменной окружения:
```powershell
$TelegramBotToken = $env:YAMUSIC_BOT_TOKEN
$TelegramChatId   = $env:YAMUSIC_CHAT_ID
```
и задавать `YAMUSIC_BOT_TOKEN` / `YAMUSIC_CHAT_ID` в системе перед запуском — тогда сам скрипт можно публиковать вообще без правок.
Структура файлов
```
YaMusicToTelegram.ps1     — сам скрипт
pause_cover.png            — картинка для паузы/простоя (обязательна, кладите рядом со скриптом)
last_message_id.txt        — создаётся автоматически: id текущего опубликованного поста
track_events.log           — создаётся автоматически: лог событий (старт/пауза/возобновление)
```
Как это работает
Раз в секунду (`$PollIntervalSeconds`) скрипт спрашивает у Windows, какой трек сейчас активен в сессии, подходящей под `$AppNameFilter`.
По статусу сессии определяется одно из трёх состояний: Playing, Paused, Idle (сессии нет или она не активна).
При смене трека (артист+название) или входе/выходе из паузы — старый пост в канале удаляется, публикуется новый, событие пишется в `track_events.log`, таймер сбрасывается (или, для паузы, «замораживается» — время на паузе не засчитывается в прогресс).
Пока трек тот же и просто играет — пост не пересоздаётся, а раз в `$CaptionUpdateIntervalSeconds` секунд у него правится подпись (обновляется прогресс-бар).
Обложка трека ищется в каталоге Яндекс.Музыки по артисту и названию — один раз при старте трека, результат кэшируется.
Для паузы/простоя используется локальный файл `pause_cover.png`: он загружается в Telegram один раз, дальше используется полученный `file_id` — повторной загрузки файла не происходит.
Устранение проблем
`429 Too Many Requests` — Telegram ограничивает частоту правок одного сообщения. Увеличьте `$CaptionUpdateIntervalSeconds` (строка 32), например до `8`–`10`.
Скрипт «зависает» — чаще всего это не зависание, а Windows-консоль ставит процесс на паузу при клике мышкой по окну (QuickEdit Mode). Скрипт отключает этот режим сам при запуске; если всё равно происходит — не кликайте по окну консоли во время работы.
Нет обложки — Яндекс.Музыка не нашла трек по запросу «артист + название» в своём каталоге (например, редкий ремикс или неправильно указан артист в самом плеере). Пост в этом случае публикуется без картинки.
«Ни одна сессия не подошла под фильтр» — Windows видит медиа-сессию, но её `SourceAppUserModelId` не содержит `yandex`. Включите `$VerboseLog = $true` (строка 28), чтобы увидеть в консоли список всех найденных сессий и поправить `$AppNameFilter` под ваш случай.
